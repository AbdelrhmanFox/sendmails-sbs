      Backend configuration (not entered here)
      
          
      The dashboard never stores Supabase keys. Add them once in Netlify for this site, then redeploy.
      
          
        
            
      • If you use Netlify’s Supabase integration, it usually sets SUPABASE_DATABASE_URL, keys, and SUPABASE_JWT_SECRET. Check “Redeploy site” when saving so functions reload.
        • 
            
      • SUPABASE_URL — Optional if SUPABASE_DATABASE_URL is present; otherwise https://<project-ref>.supabase.co.
        • 
            
      • SUPABASE_SERVICE_ROLE_KEY — Secret key (sb_secret_…). Not the publishable key.
        • 
            
      • SUPABASE_ANON_KEY — Publishable key (Live Session / realtime).
        • 
            
      • JWT_SECRET or SUPABASE_JWT_SECRET — Used to sign dashboard login tokens.
        • 
            
      • N8N_FINANCE_WEBHOOK_SECRET — Optional shared secret for the finance-data n8n report POST (header X-N8n-Secret).
        • 
          
      
          
      See .env.example for names. Local dev uses .env (gitignored).
